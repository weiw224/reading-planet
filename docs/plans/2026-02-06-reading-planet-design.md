## 1. 产品概览

### 1.1 产品名称
**阅读星球** —— 儿童阅读理解训练微信小程序

### 1.2 产品定位
面向**小学 1-6 年级**学生的阅读理解训练工具，通过每日阅读打卡、智能题目练习、勋章激励等方式，帮助孩子提升阅读理解能力。

### 1.3 核心用户
- **主要用户**：小学生（6-12岁）
- **决策用户**：家长

### 1.4 产品架构
```
┌─────────────────────────────────────┐
│          微信小程序（用户端）          │
│  登录 → 首页 → 阅读 → 答题 → 个人中心   │
└─────────────────────────────────────┘
                  ↕ API
┌─────────────────────────────────────┐
│            后台管理系统               │
│  文章管理 / 题目管理 / 用户管理 / AI导入 │
└─────────────────────────────────────┘
```

### 1.5 第一版核心功能（MVP）
1. ✅ 用户端：阅读文章 + 答题 + 每日打卡 + 勋章
2. ✅ 管理端：文章/题目管理 + AI 内容导入 + 用户数据

## 2. 内容体系

### 2.1 文章来源
- **公版读物为主**：伊索寓言、安徒生童话、格林童话、中国民间故事、唐诗宋词、经典文言文等
- **版权无忧**：均为公共领域作品，无需授权

### 2.2 内容处理策略

| 内容类型 | 处理方式 |
|---------|---------|
| 故事集（如伊索寓言） | AI 自动拆分为独立故事 → 管理员审核调整 |
| 长篇现代文 | AI 智能节选精华段落（根据难度调整长度） |
| 长篇文言文（如滕王阁序） | AI 节选适合该年级的段落 |
| 所有节选内容 | 标注出处（如"节选自《小王子》第三章"） |

### 2.3 多维度标签体系

| 标签维度 | 示例值 |
|---------|--------|
| 年级适配 | 1-6年级（推荐，非限制） |
| 文体类型 | 童话、寓言、科普、散文、古诗词、文言文 |
| 来源系列 | 伊索寓言、安徒生童话、唐诗三百首 |
| 主题标签 | 勇气、友谊、诚实、自然、科学 |
| 能力训练点 | 主旨概括、细节提取、逻辑推理等（10个维度） |
| 文化来源 | 中国传统、西方经典 |
| 改编程度 | 原文、简化版、白话改编 |
| 字数范围 | 300字内、300-500、500-800、800+ |
| 题型标签 | 选择题、填空题、判断题、简答题 |

### 2.4 难度双维度体系

```
┌─────────────────────────────────────────────┐
│               难度双维度体系                  │
├─────────────────────────────────────────────┤
│  📖 文章难度：⭐ / ⭐⭐ / ⭐⭐⭐              │
│     (基于字词、句子、篇幅、文体、抽象程度)      │
├─────────────────────────────────────────────┤
│  ❓ 题目难度：⭐ / ⭐⭐ / ⭐⭐⭐              │
│     (基于答案显隐度、推理步骤、能力维度)        │
├─────────────────────────────────────────────┤
│  🎓 适配年级：1-6年级（推荐，非限制）          │
│     低年级也可挑战高难度 ✓                    │
└─────────────────────────────────────────────┘
```

**设计理念**：年级 ≠ 难度上限，让有能力的孩子可以挑战更高难度。

## 3. 能力训练与题型

### 3.1 阅读理解能力维度（10项）

#### 一、信息获取能力
| 维度 | 说明 |
|------|------|
| **细节提取** | 找到文中明确写出的具体信息 |
| **关键信息定位** | 定位时间、地点、人物等关键要素 |

#### 二、理解与概括能力
| 维度 | 说明 |
|------|------|
| **主旨概括** | 归纳文章中心思想、主要内容 |
| **词语理解** | 结合语境理解词语含义 |
| **段落大意** | 理解段落主要内容及段落间关系 |

#### 三、分析与推理能力
| 维度 | 说明 |
|------|------|
| **人物分析** | 分析人物性格、特点、动机 |
| **情感理解** | 体会人物情感变化、作者情感态度 |
| **逻辑推理** | 根据已知信息推断隐含意思 |
| **因果关系** | 理解事件之间的因果逻辑 |

#### 四、评价与表达能力
| 维度 | 说明 |
|------|------|
| **观点表达** | 发表个人看法、评价人物或事件 |

### 3.2 题型设计（4种）

| 题型 | 适合的能力维度 | 特点 |
|------|---------------|------|
| **选择题** | 全维度适用 | 最常用，易于评分 |
| **判断题** | 细节提取、逻辑推理 | 快速检验理解 |
| **填空题** | 词语理解、细节提取 | 考察精准度 |
| **简答题（语音输入）** | 观点表达、人物分析、情感理解 | 培养表达能力，支持语音转文字 |

### 3.3 答题体验
- ✅ 答题时可随时**查看原文**（开卷模式）
- ✅ 每题有**温柔提示**按钮（引导思考方向）
- ✅ 答完显示**能力分布雷达图** + 得分

**设计理念**：阅读理解的核心是"从原文找答案"的能力，而不是死记硬背。

## 4. 用户端功能流程

### 4.1 用户旅程

```
首次使用                    日常使用
   │                          │
   ▼                          ▼
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│ 进入  │ → │ 浏览  │ → │ 登录  │ → │ 选择  │ → │ 开始  │
│小程序 │    │ 体验  │    │(需要时)│    │ 年级  │    │ 使用  │
└──────┘    └──────┘    └──────┘    └──────┘    └──────┘
                              │
                              ▼
                    自动获取微信昵称头像
```

**设计亮点**：
- **延迟登录**：先体验，需要时再登录（减少流失）
- **极简注册**：只选年级，昵称头像自动获取微信信息

### 4.2 核心页面与功能

#### 1️⃣ 首页
| 功能模块 | 说明 |
|---------|------|
| 今日成就 | 勋章数、成长值、连续天数 |
| 今日推荐 | 随机推送一篇适合年级的文章 |
| 学习月历 | 展示打卡记录，今日是否已完成 |
| 激励提示 | "再读X天即可获得XX勋章" |

#### 2️⃣ 阅读页
| 功能 | 说明 |
|------|------|
| 文章信息 | 年级、字数、预计时间、难度 |
| 文章内容 | 分段展示，重点句可高亮 |
| 出处标注 | 如"节选自《小王子》第三章" |
| 开始答题 | 阅读完成后进入答题 |

#### 3️⃣ 答题页
| 功能 | 说明 |
|------|------|
| 进度条 | 第X/Y题 |
| 题目展示 | 支持4种题型 |
| 查看原文 | 随时可回看文章 |
| 温柔提示 | 引导思考方向 |

#### 4️⃣ 完成页
| 功能 | 说明 |
|------|------|
| 得分展示 | 正确率、用时 |
| 能力雷达图 | 本次练习的能力分布 |
| 勋章解锁 | 如果触发条件则展示新勋章 |
| 继续/结束 | 再来一篇 or 今天到此为止 |

#### 5️⃣ 个人中心
| 功能 | 说明 |
|------|------|
| 用户信息 | 头像、昵称、等级、连续天数 |
| 勋章墙 | 已获得 / 未解锁的勋章 |
| 能力雷达图 | 综合能力分布 |
| 补弱项入口 | 点击进入针对弱项的文章推荐 |
| 历史记录 | 最近练习记录 |

### 4.3 文章推荐逻辑

| 场景 | 推荐逻辑 |
|------|----------|
| **首页每日推荐** | 随机推送（轻松探索） |
| **个人中心"补弱项"** | 智能推荐针对弱项能力的文章 + 题目偏向弱项 |

### 4.4 提醒机制
- 第一版采用**小程序内提示**：打开小程序时显示"今日还未打卡"

## 5. 后台管理系统

### 5.1 文章管理
| 功能 | 说明 |
|------|------|
| 文章列表 | 筛选、搜索、分页查看所有文章 |
| 新增文章 | 手动录入文章内容 |
| 编辑文章 | 修改内容、调整标签、设置难度 |
| 审核文章 | 审核 AI 导入的内容，确认/修改/拒绝 |
| 删除/下架 | 移除不合适的内容 |

### 5.2 题目管理
| 功能 | 说明 |
|------|------|
| 题目列表 | 按文章查看关联题目 |
| 新增题目 | 为文章添加题目（4种题型） |
| 编辑题目 | 修改题干、选项、答案、能力标签 |
| 审核题目 | 审核 AI 生成的题目 |
| 题目难度 | 设置题目难度星级 |

### 5.3 AI 内容导入
| 功能 | 说明 |
|------|------|
| 导入任务 | 指定书籍/来源，触发 AI 批量导入 |
| 自动拆分 | AI 将故事集拆分为独立文章 |
| 自动节选 | AI 为长文/文言文智能节选（根据难度调整） |
| 自动打标签 | AI 自动标注年级、文体、主题等 |
| 自动生成题目 | AI 为每篇文章生成配套题目 |
| 待审核队列 | 管理员审核 AI 产出内容 |

### 5.4 用户管理
| 功能 | 说明 |
|------|------|
| 用户列表 | 查看所有注册用户 |
| 用户详情 | 查看单个用户的学习数据、能力雷达 |
| 数据统计 | 总用户数、日活、打卡率等 |

### 5.5 系统配置
| 功能 | 说明 |
|------|------|
| 勋章配置 | 设置勋章名称、图标、获取条件 |
| 标签管理 | 管理年级、文体、主题等标签选项 |
| 能力维度配置 | 管理10个能力维度的名称、描述 |

## 6. 激励体系

### 6.1 每日打卡
| 项目 | 说明 |
|------|------|
| 完成条件 | 完成 1 篇文章阅读 + 答题（不论对错） |
| 展示方式 | 学习月历显示打卡状态 |
| 连续天数 | 首页显示"🔥 连续X天" |

**设计理念**：降低门槛、培养习惯比追求短期正确率更重要。

### 6.2 勋章系统

| 类别 | 勋章示例 | 获取条件 |
|------|---------|---------|
| **坚持类** | 阅读新星 | 首次完成阅读 |
| | 三日小达人 | 连续打卡3天 |
| | 周冠军 | 连续打卡7天 |
| | 月度之星 | 连续打卡30天 |
| **能力类** | 细节大师 | 细节提取正确率达90% |
| | 推理达人 | 逻辑推理题累计答对20题 |
| | 表达之星 | 完成10道语音简答题 |
| **阅读量类** | 小书虫 | 累计阅读10篇 |
| | 博学多才 | 累计阅读50篇 |
| | 阅读狂人 | 累计阅读100篇 |
| **探索类** | 童话爱好者 | 阅读10篇童话 |
| | 诗词达人 | 阅读10篇古诗词 |
| | 全能选手 | 每种文体都阅读过 |

### 6.3 激励提示
- 首页底部显示：**"继续保持！再读X天即可获得XX勋章"**
- 完成练习后若解锁新勋章，弹窗庆祝展示

### 6.4 后续版本可扩展（暂不实现）
- 📈 等级系统（Lv.1 → Lv.99）
- 🏆 排行榜（超越X%同学）
- 👨‍👩‍👧 家长端数据报告

## 7. 技术架构

### 7.1 技术栈

```
┌─────────────────────────────────────────────────────────┐
│              微信小程序（原生开发）                        │
└─────────────────────────────────────────────────────────┘
                           │ HTTPS
                           ▼
┌─────────────────────────────────────────────────────────┐
│                  Python + FastAPI                        │
│              数据库：PostgreSQL / MySQL                   │
│              AI：硅基流动 Qwen3-8B API                    │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│              管理后台：Vue 3 + Element Plus               │
└─────────────────────────────────────────────────────────┘
```

### 7.2 技术选型说明

| 层级 | 技术选择 | 选择理由 |
|------|---------|---------|
| 小程序 | 原生微信开发 | 性能最优，官方支持完善 |
| 后端框架 | FastAPI | 高性能异步，自动生成API文档，适合API服务 |
| 数据库 | PostgreSQL/MySQL | 成熟稳定，支持复杂查询 |
| 管理后台 | Vue 3 + Element Plus | 轻量高效，中文生态好 |
| AI 服务 | 硅基流动 Qwen3-8B | 免费额度适合前期，中文能力好 |

## 附录：设计决策记录

| 决策点 | 决策结果 | 理由 |
|--------|---------|------|
| 打卡条件 | 完成1篇即可，不限正确率 | 降低门槛，培养习惯 |
| 答题能否看原文 | 可以随时查看 | 阅读理解是找答案能力，非记忆 |
| 推荐逻辑 | 首页随机，补弱项智能 | 首页轻松，专项训练针对性 |
| 年级与难度关系 | 年级是推荐，非限制 | 允许挑战，成长思维 |
| 登录时机 | 延迟登录 | 先体验，减少流失 |
| 技术栈 | 原生小程序 + FastAPI + Vue | 性能优、开发效率高 |
